DOC2MD / LANDSCAPE
No. 03 / 12
定位議題 / Not a slam dunk

MarkItDown 很優秀 — 前提是用對場景。

MarkItDown is excellent for the documents it was built for.

適合的場景 Where it shines

RIGHT JOB
  • 純文字流的 .docx memo
  • .xlsx / .pptx — 結構已在檔案裡
  • 多格式批次 ingestion（email、HTML、圖）
  • 快速建一份混合格式 RAG corpus

不適合的場景 Where it breaks

WRONG JOB
  • JEDEC / T10 規格書中的 register 表
  • timing diagram、state machine、waveform
  • 多欄排版的 typeset 頁面
  • 表格本身就是內容的文件

本專案仍然裝了 MarkItDown — 它有它的位置。對 spec 我們帶更重的武器。

我們的做法 / Our approach

Marker 打底。視覺模型收拾剩下的。

Marker does the bulk. A vision model fixes the rest.
LAYER 01
Marker OCR 基底
對每一頁都產出 Markdown。分頁輸出保留 per-page provenance， 下游可精準針對需要補強的頁面處理。
~15s / page · CPU-capable · keeps images · weak on complex tables
LAYER 02
視覺模型補強
規則掃描標記問題頁（亂碼、空表、缺圖）， 這些頁再交給視覺模型 — Claude、Anthropic API、或公司地端 AI。
full-page redo + per-figure descriptions for RAG retrieval

關鍵在於 — 視覺模型是誰，可以配置。 這就是兩種模式的由來。

一套流程，兩種分派 / Same pipeline, two dispatches

依文件性質，選擇不同的視覺模型。

Pick the vision model that matches the document.

CLI Mode For public documents

公開文件
  • 零外部 API 呼叫，資料不離開本機
  • 當前 Claude Code session 就是視覺模型
  • pipeline render PNG、產 manifest、暫停、交棒
  • Claude 讀 PNG 寫 Markdown，pipeline 繼續
  • 成本：零

API Mode For confidential documents

機密文件
  • 可替換 provider：Anthropic / OpenAI-compatible
  • 對接公司地端 AI（Qwen / LLaMA / 自有 gateway）
  • 完全自動，不需 session 介入
  • 文件從不離開企業邊界
  • 成本：依服務商；地端通常 ~0
06
DOC2MD / TWO MODES
No. 06 / 12
Mode A · 公開文件 / Public documents

CLI Mode — session 本身就是那個模型。

The Claude Code session becomes the vision model.
  1. Marker 轉換每頁；規則掃描標記 N 頁問題。
  2. Pipeline 把問題頁 render 成 PNG，放進 pages_for_vision/。
  3. 寫入 need_vision.json 任務清單，然後乾淨退出。
  4. Claude Code session 讀 manifest、看每張 PNG、把 Markdown 寫回 marker_cache/page_NNNN.md。
  5. 操作者以 --resume 再跑；pipeline 驗證每個 task 都寫完，然後合併章節。
  6. 章節 description 階段（Step 7）同樣暫停-繼續機制。
$python -m doc_to_markdown.main input/spec.pdf --mode cli

DOC2MD / CLI FLOW
No. 07 / 12
Mode B · 機密文件 / Confidential documents

API Mode — 一行環境變數，切到自家 AI gateway。

A one-line swap to your on-prem AI endpoint.

同一份原始碼。Anthropic、OpenAI-compatible 地端 AI、或任何相容 gateway — 只改環境變數。 

# 公司地端 AI（Qwen / LLaMA / 內部 gateway）
export VISION_PROVIDER=openai-compatible
export VISION_BASE_URL=https://your-company-ai.internal/v1
export VISION_API_KEY=$TOKEN
export VISION_MODEL=qwen-vl-max

python -m doc_to_markdown.main input/confidential.pdf --mode api

改回 VISION_PROVIDER=anthropic 即是公開 Anthropic API。 對接內部 endpoint 時，文件從不離開企業邊界。

09
DOC2MD / PIPELINE
No. 09 / 12
品質控制 / Quality control

兩層保障 — 一層擋不住的那層。

Two layers catch what one cannot.
LAYER 01·規則掃描 / Rules

快速、免費、確定性 Fast, free, deterministic

  • 亂碼字元比例 > 5%
  • 有表頭但無資料列
  • 頁面有圖片，Markdown 卻沒有
  • 內容長度 < 50 字元
  • 格式破損（未閉表、孤立標題）
LAYER 02·抽樣比對 / Sampling (api only)

分類抽檢 Category spot-check

  • 按 table / image / text 類別比例抽樣
  • Vision 比對 MD 與頁面圖像，回傳 0-1 相似度
  • 低於 0.70 標記為問題頁
  • 若某分類失敗率 > 50%，整類重做

CLI Mode 跳過需要 API 的抽樣層。規則層已擋下 ~95% 的問題，剩下由視覺補強收尾。
